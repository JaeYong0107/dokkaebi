# dev → main 머지 전 Preview 체크리스트

`dev` 브랜치 14건 기능 구현 후 Preview 배포에서 **한 번에 쭉 훑는** 검증
절차. 모든 항목이 ✅ 되면 안전하게 main 머지 가능.

## 준비

### Preview URL 찾기

Vercel Dashboard → **Deployments 탭** → 최신 `dev` 브랜치 배포 (Preview 배지)
→ 클릭 후 상단 URL 복사. 형태:

```
https://dokkaebi-git-dev-jaeyongs-projects-<hash>.vercel.app
```

아래 **[PREVIEW_URL]** 로 표기.

### 테스트 계정

| 역할 | 이메일 | 비밀번호 |
|---|---|---|
| 관리자 | admin@dokkaebi.kr | admin1234 |
| 일반 고객 | user@dokkaebi.kr | user1234 |
| 사업자 (승인됨) | biz@dokkaebi.kr | biz12345 |

⚠️ **주의**: Preview 는 프로덕션과 **같은 Neon DB 를 씀.** 쓰기 동작(회원가입·
결제·사업자 신청 등)은 실제 DB 에 저장됩니다. 테스트 데이터는 필요 시 수동 정리.

## Phase 1 — 비로그인 공개 페이지

### 1-1. 홈 `/`
- [ ] 상품 카드 그리드 정상 렌더
- [ ] 상단 아이콘들이 **그림** 으로 보임 (글자로 보이면 폰트 문제)
- [ ] 맨 아래 "사업자 안내" 섹션의 **"단가표 다운로드"** 클릭 → "로그인 후
  다운로드할 수 있습니다" 안내 배너

### 1-2. 상품 목록 `/products`
- [ ] 상품 12개 노출 + 하단 **"상품 더보기 (12 / 16)"** 버튼
- [ ] 더보기 클릭 → URL 에 `?page=2` 추가 + 전체 16개 노출 + **"모든 상품을
  다 불러왔습니다"** 안내 문구
- [ ] 카테고리 칩 "과일" 클릭 → 과일만 필터링
- [ ] 상단 내비 **"사업자 전용"** 탭 클릭 → `?dealsOnly=1` 로 이동, 특가/단독/
  BEST 뱃지 상품만 노출 (수박·감자 2건 예상)

### 1-3. 상품 상세 `/products/prod-lettuce-001`
- [ ] 하단 탭 3개 **클릭 시 내용 전환**:
  - "배송/교환/환불 정보" → 배송·반품 안내 텍스트
  - "상품 문의" → "문의 기능은 곧 오픈 예정" 플레이스홀더
  - "구매 리뷰" → "첫 리뷰는 구매 완료 후..." 플레이스홀더

### 1-4. 가드 확인
- [ ] `/mypage` 접속 → `/login?callbackUrl=%2Fmypage` 리다이렉트
- [ ] `/admin` 접속 → 동일하게 `/login` 리다이렉트

## Phase 3 — 사업자 쇼핑 (biz@dokkaebi.kr)

### 3-1. 도매가 확인
- [ ] 로그인 후 상품 상세 열면 **사업자 할인가** 자동 표시
- [ ] `/products/prod-lettuce-001` → 14,430원 (22% 할인)

### 3-2. 결제 배송지 변경 (신규)
- [ ] 장바구니 담고 `/checkout` 이동
- [ ] "배송지 정보" 섹션 오른쪽 **"배송지 변경"** 클릭 → 모달
- [ ] 받는 분 / 연락처 / 주소 / 상세 입력 → 저장 → 블록에 반영
- [ ] 배송 요청사항 select 값 바꿔도 state 유지 (새로고침 시 localStorage 에서 복원)
- [ ] 주문하기 → 결제 완료 → `/orders/complete`

### 3-3. 배송 조회 + 추가 기능
- [ ] `/orders` → 방금 주문 포함 목록
- [ ] 주문 클릭 → tracking 페이지
- [ ] **"영수증 출력"** 클릭 → 브라우저 인쇄 다이얼로그
- [ ] **"고객센터 문의"** 클릭 → mailto + 주문번호 자동 채움
- [ ] **배송지 정보 "변경하기"** 클릭:
  - PAID 상태 주문 → 모달 열림, 저장 → 반영
  - SHIPPING 상태 시드 주문은 "변경하기" disabled (이미 출고)

### 3-4. 홈 단가표 다운로드 (승인 사업자)
- [ ] 홈으로 가서 **"단가표 다운로드"** 클릭 → `dokkaebi-price-list-YYYYMMDD.csv`
  파일 다운로드
- [ ] 엑셀에서 열면 한글 깨짐 없이, 정가·일반가·사업자가 3열 모두 채워져 있음

### 3-5. 홈 상담 신청
- [ ] **"상담 신청하기"** → `mailto:sales@dokkaebi.kr` + "대량 주문 수량/...
  양식 자동 채움

## Phase 4 — 관리자 (admin@dokkaebi.kr)

### 4-1. 대시보드 `/admin`
- [ ] 상단 **알림 벨 아이콘** 우상단에 **빨간 배지** (저재고 상품 수)
- [ ] 벨 클릭 → 드롭다운
  - "사업자 승인 대기" 카드 → 클릭 → `/admin/users?approval=PENDING&type=BUSINESS`
    이동 (Phase 2-2 에서 신청한 user 계정 보이는지)
  - "저재고 상품" 카드 → 클릭 → `/admin/products?active=ACTIVE`
- [ ] 대시보드 돌아와서 "최근 주문 요약" 섹션 **"전체보기"** 클릭 →
  `/admin/orders`

### 4-2. 재고 추가
- [ ] 대시보드 "재고 부족 알림" 카드의 첫 항목 `+` 아이콘 클릭 → 모달
- [ ] 추가할 수량 "10" 입력 → 저장 → 카드에 잔여 수량 반영
- [ ] 새로고침 후 영구 반영 확인

### 4-3. 사용자 관리
- [ ] `/admin/users` → Phase 2-2 에서 신청한 user@ 계정이 "BUSINESS / 대기" 로 표시
- [ ] "승인" 버튼 클릭 → 뱃지 "승인됨" 으로 전환
- [ ] 이 상태에서 user 로 다시 로그인해 마이페이지가 "사업자 혜택 활성" (초록) 배너인지
- [ ] (검증 후 관리자로 돌아와) 승인 해제 후 원래 NORMAL 로 되돌리기 — Neon SQL 에서
  `UPDATE "User" SET customerType='NORMAL', businessApproved=false, businessName=NULL,
  businessNumber=NULL WHERE email='user@dokkaebi.kr';` 로 정리

### 4-4. 주문 CSV 다운로드
- [ ] `/admin/orders` → 상단 **"CSV 다운로드 (N)"** 클릭
- [ ] 파일 `dokkaebi-orders-YYYYMMDD-HHMM.csv` 받아서 엑셀에서 열기
- [ ] 15열 헤더 (주문번호~택배사), 한글 깨짐 없음, 현재 필터 결과만 포함

### 4-5. 상태 필터와 CSV 연동
- [ ] `/admin/orders?status=PAID` 이동 → 다시 CSV 다운로드
- [ ] 버튼 라벨 카운트와 CSV 행 수 일치
